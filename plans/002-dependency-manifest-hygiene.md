## Why this matters

Four manifest/reality mismatches, all verified:

1. `pyproject.toml:17` pins `pydantic-ai>=0.0.14`, but `uv.lock` resolves
   **2.0.0** and the code is written against the 2.x API
   (`src/fwbg_agents/tools/llm.py:92-93` documents the 2.0 `usage` property;
   `from pydantic_ai.providers.anthropic import AnthropicProvider` is 2.x
   surface). With Dependabot automerge enabled, a breaking 3.x release can
   auto-merge and take down every agent; conversely a constraint resolve
   could legally pick a pre-2.0 version the code no longer supports.
2. `anthropic` is imported directly (`src/fwbg_agents/tools/llm.py:11`
   `from anthropic import AsyncAnthropic`; `src/fwbg_agents/tools/api_errors.py:15`
   `import anthropic`) but is **not declared** — it arrives only transitively
   via `pydantic-ai-slim[anthropic]`. If pydantic-ai reshuffles its extras,
   the model factory and the error-classification path break at import time.
3. `tavily-python>=0.5.0` (`pyproject.toml:20`) is never imported — the real
   Tavily integration is a hand-rolled httpx client in
   `src/fwbg_agents/tools/search/tavily.py`.
4. `structlog>=24.4.0` (`pyproject.toml:22`) is never imported — all logging
   is stdlib `logging`.

## Steps

### Step 1: Tighten the pydantic-ai constraint

In `pyproject.toml`, change `"pydantic-ai>=0.0.14"` → `"pydantic-ai>=2.0,<3"`.

**Verify**: `uv lock` → exit 0; `git diff uv.lock | head` shows no version
change for pydantic-ai itself (still 2.0.x).

### Step 2: Declare `anthropic` explicitly

Add `"anthropic>=0.40"` to `[project] dependencies` (the locked transitive is
0.111.0; `>=0.40` matches what `pydantic-ai-slim[anthropic]` itself allows —
check with `grep -B2 -A8 'name = "pydantic-ai-slim"' uv.lock` and use its
anthropic floor if higher).

**Verify**: `uv lock && uv sync` → exit 0; `uv tree | grep -i "^anthropic\| anthropic"` shows anthropic as a direct dependency.

### Step 3: Remove `tavily-python` and `structlog`

Delete both lines from `[project] dependencies`.

**Verify**: `uv lock && uv sync` → exit 0;
`grep -rn "import tavily\|from tavily\|import structlog" src/ scripts/ tests/` → still no matches;
`uv run pytest -q` → all pass.

### Step 4: Shield pydantic-ai from automerged majors

If `.github/dependabot.yml` exists, add under the pip/uv ecosystem entry:

```yaml
    ignore:
      - dependency-name: "pydantic-ai"
        update-types: ["version-update:semver-major"]
```

If the file does not exist (automerge may be configured repo-side only),
skip this step and record that in the commit message body.

**Verify**: `uv run pytest -q` → all pass (final gate).
